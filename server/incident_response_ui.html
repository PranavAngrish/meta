Frequently Asked Questions
How is the score calculated?
Your score uses 6 weighted dimensions: Root Cause (30%), Remediation (25%), Investigation (15%), Efficiency (10%), Safety (10%), and Sequence (10%). Scores are clamped to the open interval (0.01, 0.99).
What is an "Observation Loop"?
If you perform 3+ diagnosis actions in a row without any fix action, the system detects an observation loop. Your score is hard-capped at 0.45 and you receive a -0.08 penalty. Mix investigation and fix actions!
What is "Collateral Degradation"?
Every 4 steps, if the incident isn't resolved, dependent services begin degrading — error rates rise 10%, latency climbs 40%. Eventually the whole stack fails. Time pressure is real and represents real incident dynamics.
What's a "Diagnosis Gate"?
Some scenarios require investigating a service BEFORE you can successfully fix it. If you apply a fix blind, you only receive 50% of the remediation points. Investigate first, then fix.
What does a "Seed" do?
Seeds make scenarios reproducible. Same seed = same incident with identical metric values. Different seed = same scenario structure but metric values shift ±12%. Leave blank for a random variation. Use seed=42 for testing.
How does Alert Triage differ?
Alert Triage is a different task type — instead of fixing an incident, you classify its severity (P1 Critical → P4 Low). You only have 3 steps. Score is based on severity accuracy plus an investigation bonus for checking multiple services.
How do I improve my score?
Investigate thoroughly across services, match your root cause declaration exactly to the expected cause, apply the correct corresponding fix, declare root cause BEFORE fixing, and avoid repeating the same investigation action.
